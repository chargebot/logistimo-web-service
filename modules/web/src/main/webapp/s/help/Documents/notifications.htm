Notifications

To view and manage notification configuration, perform the following steps:

1.  Go to Configuration > Notification

Specify the Specify the kinds and formats of notification messages sent via SMS or email on various events in the system. Each message can use the mutiple variables in a sentence, as shown below (variables are case-sensitive). For SMS messages, along with the variables, the message length should not exceed 160 characters. Variables and their expected character lengths are shown below. 

2.  Every event that happens in Logistimo is shown in this view.

3.  The Event notification tab shows events in orders, inventory, accounts and setup.

4.  Create order notifications, as briefed below:

•      To create order notifications go to Notification > Orders , select the event notification you wish from the list, such as:

i. Fulfillment due: to notify the order before which the fulfillment is due.

ii.          No order activity: to notify since when the particular order has no activity.

iii.         Order created: to notify a new order.

iv.         Order expired/ untouched: to notify orders expired or untouched.

v.         Order modified: to notify modification of existing orders.

vi.         Order status Changed: to notify the status change of an order (cancelled, confirmed, shipped or fulfilled)

vii.        Payment made on order: to notify the payment made.

•      To create inventory notifications go to Notification > Inventory, select the event notification you wish from the list, such as:

i. Batch expiry: to notify inventory expired by batch.

ii.          No inventory activity: to notify since when the particular inventory has no activity.

iii.         Out of stock: to notify the inventory out of stock.

iv.         Stock count differs from current stock: to notify the % of difference from the stock count to current stock.

v.         Stock count: to notify the reason for stock count (retrial, routine or new-employee).

vi.         Stock issued: to notify the reason for stock issued (free, promotion, sale or production).

vii.        Stock transferred: to notify the reason for stock transferred (emergency or excess).

viii.        Stock received: to notify the reason for stock received (transfer, return or stock out).

ix.         Stock back from normal level: to notify the replenishment of abnormal stock.

x.         Stock discarded: to notify the reason for stock discarded (damaged, expired, frozen or VVM unusable)

•      To create account notifications go to Notification > Accounts, select the event notification you wish from the list, such as:

i. Credit limit exceeded: to notify if the credit limit has exceeded

•      To create setup notifications go to Notification > Setup, select the event notification you wish from the list, such as:

i. Entity created, modified or deleted.

ii.          Inventory created or deleted.

iii.         Material created, modified or deleted.

iv.         No user logins.

v.         User created, modified or deleted.

vi.         User dormant/ inactive: to notify since when the user has been inactive.

vii.        User login IP address matched: to notify when IP address match, start typing the IP address to get the particular match.

5.  Click Add Notification.

6.  To view bulletin board, go to Notifications > Bulletin board.

•      Specify the bulletin board configuration setup, such as

i. Enable: select the option you wish by either disabling it, only for the people in your domain to see it or if it should be seen by everybody.

ii.          Data options: select the duration, maximum number of  items on board and when you wish to refresh it.

iii.         Display options: select the interval between scrolls and the slide show navigator.

7.   Click Save or Post to board.
